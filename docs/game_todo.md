# SRPG 完成までのタスク

最終更新: 2026-07-04

このファイルは、チャットの流れで出た要望をゲーム完成までの作業一覧としてまとめたものです。  
優先度は `P0` が最優先、`P1` が次点、`P2` が後回しです。

## P0: まず完成させる

- [ ] 移動キャンセルを完全に安定させる
  - 移動後でも元の位置に戻れる
  - キャンセル対象の状態を明確に保持する
  - 画面遷移や情報表示をまたいでも戻せる

- [ ] ユニット選択と情報表示の基本導線を固める
  - 味方をタップしたら情報表示
  - 敵をタップしたら情報表示
  - 情報表示から移動、攻撃、スキルに入る
  - 選択直後に勝手に移動モードへ入らない

- [ ] タイルタップ時の情報表示を安定させる
  - 地形名、座標、移動コストを表示
  - 高低差、遮蔽、水地などの情報を表示
  - マーカークリックとの挙動を統一する

- [ ] 敵の攻撃導線を完成させる
  - 攻撃可能範囲なら攻撃ボタンを表示
  - 敵情報からそのまま攻撃できる
  - 範囲外や対象外の表現を整理する

- [ ] ターン終了後の状態破綻をなくす
  - 自軍が消えたように見える問題を修正する
  - 敵が以前の位置に見える問題を修正する
  - 選択中ユニット、選択中タイル、情報表示を正しく維持する

- [ ] 敵移動の瞬間移動感を解消する
  - 1マスずつのアニメーションにする
  - 行動結果と描画を分離する
  - 移動前後の状態変化が追えるようにする

- [ ] マップの欠けている BOX を埋める
  - 中途半端に穴あきに見える箇所を修正する
  - 地形の高さ表現と描画順を見直す
  - 3D っぽい地面表現の破綻を減らす

## 実装方針メモ

- ルールや戦闘計算はできるだけ Rust / WASM 側へ寄せる
- JavaScript は入力、表示、UI 切り替えに寄せる
- 1回の変更ごとに localhost で動作確認する
- 変更は `main` に直接反映する
