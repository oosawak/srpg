# SRPG アーキテクチャ方針

このリポジトリでは、ゲームの判断ロジックはできるだけ Rust / WASM 側に寄せる。
JS は描画、入力、DOM 操作、WASM との接続に限定する。

## WASM 側に置くもの

- ターン進行
- フェーズ管理
- 移動範囲の計算
- 攻撃範囲の計算
- 命中やダメージ計算
- AI の判断
- 地形コスト
- 視界や遮蔽の判定
- マップの投影座標
- ゲーム状態のシリアライズ

## JS 側に置くもの

- Canvas 描画
- DOM UI
- ボタンやキーボード入力の受付
- WASM とのブリッジ
- 見た目の演出

## 迷ったときの基準

- ルールに関わるなら WASM 側
- 表示だけなら JS 側
- ルールに影響しない補助機能なら、実装が楽な方を優先してよい
- どちらにも置けるなら、まず WASM 側に置く
- JS にはゲームの正しさを持たせない

## このプロジェクトで避けること

- 重要なゲームロジックを JS に逃がすこと
- JS 側で状態の真実を二重管理すること
- 画面上の見た目のためにルールを歪めること
