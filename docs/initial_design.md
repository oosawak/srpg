# SRPG 初期設計メモ

このドキュメントは、戦国 SRPG の初期設定と実装方針をまとめたものです。
Rust 側のゲームコア設計を先に固め、後から WASM / JS の表示層をつなぐ前提です。

## 1. コンセプト

- 戦国時代を題材にした SRPG
- `FFT` 系のアビリティ構造を戦国向けに再解釈する
- Rust / WASM 側でルール、AI、ターン管理を担当する
- JS 側は描画と入力に寄せる

## 2. 地形

地形はタイル単位で管理する。

```rust
struct Tile {
    terrain: Terrain,
    height: i32,
    cover: bool,    // 忍者・伏兵用
    water: bool,    // 川・田んぼ
}
```

### 地形特徴

- 山城: 高低差 + 狭い道
- 城下町: 建物 + 路地
- 田んぼ: 移動コスト増
- 段々畑: 高さ + 視界変化

## 3. ユニット

武将と兵種をまとめてユニットとして扱う。

```rust
struct Unit {
    id: UnitId,
    clan: ClanId,
    job: Job,           // 足軽・弓・槍・忍・鉄砲・軍師
    stats: Stats,
    traits: Vec<Trait>, // 武将固有特性
    abilities: Vec<Ability>,
    pos: (i32, i32),
}
```

### 兵種例

- 足軽: 押し込み、陣形維持
- 槍: 間合い反撃
- 弓: 曲射、遠距離狙撃
- 鉄砲: 火縄チャージ、三段撃ち
- 忍者: 跳躍、煙玉、隠密
- 軍師: 鼓舞、陣形変更、士気操作

## 4. アビリティ

戦国らしさを出すための特殊行動。

- 押し込み（足軽）
- 曲射（弓）
- 火縄チャージ（鉄砲）
- 煙玉（忍者）
- 陣形変更（軍師）

FFT のアビリティ構造をそのまま戦国に移植できる。

## 5. AI

Rust で高速に実装できる戦術ロジックを想定する。

### 戦術例

- 釣り野伏せ: 弱い部隊で誘導し伏兵で挟撃する
- 三段撃ち: 鉄砲隊のチャージローテーション
- 挟撃: 左右から同時攻撃してダメージ補正を得る
- 退き口: 撤退しながら射撃する

## 6. ターン制ステートマシン

ターン進行は状態機械で管理する。

```rust
enum BattleState {
    StartTurn(UnitId),
    SelectAction,
    ExecuteAction,
    EndTurn,
}
```

- Rust 側で状態管理する
- JS 側は状態に応じて UI を切り替える

## 7. 表示

WASM 側の表示は描画中心にする。

- 等角投影: FFT 風
- 真上視点: FE 風
- WebGL2 で描画
- Rust からレンダーデータを渡す
- 表示モードは `isometric` / `topdown` を切り替え可能にする
- 移動や攻撃などの判断は JS に逃がさず WASM 側に置く

## 8. 最小プロトタイプ構成

### Rust

- マップ
- ユニット
- ターン制
- AI
- アビリティ計算

### JS

- WebGL 描画
- 入力
- UI

### WASM

- `update`
- `get_render_data`
- `apply_input`

## 9. 実装優先順位

1. Rust 側の SRPG コア
2. マップ生成・読み込み
3. アビリティ
4. AI
5. JS / WASM 表示
